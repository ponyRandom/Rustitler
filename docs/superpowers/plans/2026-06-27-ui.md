# UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the frontend module described by `doc/tasks/ui.md`.

**Architecture:** Keep the UI split into focused units: IPC command wrappers, event/drop subscriptions, small observable stores, and React views that consume store snapshots. The frontend mirrors backend DTOs from `src/types/ipc.ts`, merges streamed `batch-event` payloads into local state, can repair state by refetching snapshots, and exposes main queue, detail, history, undo, and settings workflows.

**Tech Stack:** React 18, TypeScript, Vite, Tauri v2 JavaScript APIs, Vitest, Testing Library, and CSS modules via a single app stylesheet.

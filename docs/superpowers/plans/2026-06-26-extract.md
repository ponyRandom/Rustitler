# Extract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the extraction module boundary from `doc/tasks/extract.md` without coupling default debug builds to OCR/PDF native dependencies that are only validated under feature-gated spike paths.

**Architecture:** Keep `src-tauri/src/extract.rs` responsible for dispatch, common DTO construction, Word paragraph selection, DOC conversion orchestration, coordinate normalization, OCR/PDF fallback decisioning, batch temp directory management, and diagnostics save calls. Use small traits (`DocxTextExtractor`, `DocConverter`, `PdfTextExtractor`, `OcrExtractor`, `PdfRasterizer`) so default tests use deterministic fakes; real dependency adapters can be wired behind features once packaging/offline constraints are resolved.

**Tech Stack:** Rust standard library, existing `models.rs` DTOs, existing `diagnostics::Diagnostics`, existing `settings::Settings`, `tempfile`, and optional dependency adapters for `liteparse`, `undoc`, and `tesseract-rs` behind non-default features.

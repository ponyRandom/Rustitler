# History Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the SQLite history, duplicate detection, settings snapshot, and undo safety workflow described by `doc/tasks/history.md`.

**Architecture:** Keep the module self-contained in `src-tauri/src/history.rs`, backed by `rusqlite` and the existing DTOs in `models.rs`. Store normalized file results, candidates, rule details, settings snapshots, and undo metadata in SQLite; expose small functions that later `ingest`, `batch_scheduler`, and `commands` can call.

**Tech Stack:** Rust, `rusqlite` bundled SQLite, existing `AppError`/DTOs, `sha2` for undo content hashing, and `tempfile` for tests.

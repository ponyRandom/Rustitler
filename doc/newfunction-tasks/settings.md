# Settings Tasks

## 分类配置持久化任务

目标文件：`src-tauri/src/settings.rs`

- [ ] NF-SE-01 在 `Settings` 中新增 `classification_settings: ClassificationSettings` 字段，并保持旧设置文件可加载（依赖：NF-MO-01）
- [ ] NF-SE-02 实现默认分类配置：`请示`、`报告`、`通知`、`标准`，每个分类默认包含同名关键词（依赖：NF-SE-01）
- [ ] NF-SE-03 实现分类名称清洗：替换 Windows/macOS 路径非法字符、去除首尾空白、压缩连续空白（依赖：NF-SE-01）
- [ ] NF-SE-04 实现关键词清洗：去除首尾空白、压缩连续空白、替换路径非法字符（依赖：NF-SE-03）
- [ ] NF-SE-05 校验普通分类名称清洗后不能为空，失败时返回结构化设置错误（依赖：NF-SE-03）
- [ ] NF-SE-06 校验普通分类至少包含一个清洗后非空关键词（依赖：NF-SE-04）
- [ ] NF-SE-07 校验清洗后的分类名称不能重复，重复时不自动追加序号或合并（依赖：NF-SE-05）
- [ ] NF-SE-08 校验清洗后的关键词不能重复，且同一关键词不能属于多个普通分类（依赖：NF-SE-06）
- [ ] NF-SE-09 允许 `其他` 和 `待确认` 从保存配置中缺失，运行时由分类模块兜底重建（依赖：NF-SE-02）
- [ ] NF-SE-10 保存设置时复用现有原子写入流程，并写入清洗后的分类配置（依赖：NF-SE-03 至 NF-SE-08）
- [ ] NF-SE-11 添加设置测试，覆盖默认值、旧文件兼容、清洗、空值、分类名重复、关键词重复和系统分类缺失（依赖：NF-SE-10）
- [ ] NF-SE-12 运行设置测试：`rtk cargo test --manifest-path src-tauri/Cargo.toml settings --lib`（依赖：NF-SE-11）

验收点：

- 分类配置只保存在本机设置中。
- 第一版不接入设置导入/导出扩展能力。

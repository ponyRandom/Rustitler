# Rustitler 文件夹分类功能需求文档

版本：0.1  
日期：2026-07-03  
状态：需求确认稿  
目标功能：按文件名规则递归分类文件夹中的文件

## 1. 背景

Rustitler 当前已经支持离线导入 PDF、DOCX、DOC、图片等文档，提取正文或版式信息，识别最像“标题”的文本，并将原文件复制到 `Rustitler 输出` 目录后按标题重命名。低置信度结果可由用户手动确认，应用也支持历史记录和撤销输出。

新功能不是对现有标题识别流程的简单补充，而是一个独立的“文件夹分类复制”工具。用户选择一个源文件夹后，Rustitler 递归扫描该文件夹及其子目录中的文件，根据文件当前名称进行分类，并把文件复制到源文件夹同级的分类输出目录中。源文件夹内容保持不变。

## 2. 功能目标

- 用户可以在主界面发起“分类文件夹”操作。
- 用户选择一个源文件夹后，系统递归扫描其中所有子目录里的普通文件。
- 系统根据文件名主体匹配分类关键词。
- 分类结果复制到源文件夹同级的新输出目录。
- 输出目录按批次新建，避免和历史分类结果混在一起。
- 文件复制失败不影响其他文件继续处理。
- 分类完成后展示本次摘要。
- 第一版完全离线运行，不使用 AI、云端服务或正文语义判断。

## 3. 非目标

第一版不包含以下能力：

- 不读取正文内容做分类。
- 不调用现有标题提取、评分、重命名流程。
- 不修改、移动或删除源文件。
- 不保留源文件夹下的子目录层级。
- 不做执行前分类预览。
- 不写入现有历史页面。
- 不支持撤销分类结果。
- 不支持取消正在执行的分类批次。
- 不将分类配置纳入现有设置导入/导出。

以上能力可作为后续扩展方向记录。

## 4. 用户流程

```text
用户点击主界面“分类文件夹”
  -> 选择源文件夹
  -> 后端递归扫描源文件夹
  -> 跳过隐藏文件和系统文件
  -> 按文件名主体匹配分类关键词
  -> 创建同级输出目录
  -> 创建分类子目录
  -> 复制文件到分类目录
  -> 返回分类摘要
  -> 前端展示摘要
```

示例：

```text
输入文件夹/
  关于预算调整的请示.pdf
  子目录/
    会议通知.png

Rustitler 分类输出 2026-07-03 1530/
  请示/
    关于预算调整的请示.pdf
  通知/
    会议通知.png
```

## 5. 输入与输出

### 5.1 输入

输入是用户选择的一个源文件夹。源文件夹可以是任意名称，不要求必须叫 `Rustitler 输出`。

扫描规则：

- 递归扫描源文件夹及所有子目录。
- 普通文件参与分类。
- 隐藏文件和系统文件不参与分类。
- 隐藏文件和系统文件不计入完成摘要。
- 支持格式和非支持格式都会参与输出。

当前 Rustitler 支持格式包括：

- `.docx`
- `.doc`
- `.pdf`
- `.png`
- `.jpg`
- `.jpeg`

非支持格式也会被复制，但只能进入 `其他` 分类。

### 5.2 输出

输出目录创建在源文件夹同级，目录名格式为：

```text
Rustitler 分类输出 YYYY-MM-DD HHmm
```

示例：

```text
Rustitler 分类输出 2026-07-03 1530
```

每次执行分类都创建新的时间戳输出目录。即使同级已有旧的 `Rustitler 分类输出 ...` 目录，也不复用旧目录。

如果同一分钟内连续执行分类导致输出目录名冲突，系统自动追加序号，绝不覆盖已有输出目录：

```text
Rustitler 分类输出 2026-07-03 1530
Rustitler 分类输出 2026-07-03 1530 (2)
Rustitler 分类输出 2026-07-03 1530 (3)
```

输出结构：

```text
Rustitler 分类输出 YYYY-MM-DD HHmm/
  请示/
  报告/
  通知/
  标准/
  其他/
  待确认/
```

输出不保留源目录层级。源文件无论来自源文件夹根目录还是深层子目录，都会被复制到对应分类目录的第一层。

## 6. 分类规则

### 6.1 默认分类

默认普通分类与关键词：

| 分类目录 | 默认关键词 |
| --- | --- |
| 请示 | 请示 |
| 报告 | 报告 |
| 通知 | 通知 |
| 标准 | 标准 |

系统保底分类：

| 分类目录 | 用途 |
| --- | --- |
| 其他 | 未命中任何分类，或非支持格式 |
| 待确认 | 文件名命中多个分类 |

### 6.2 匹配规则

- 只匹配文件名主体，不包含扩展名。
- 匹配不区分大小写。
- 文件名命中一个分类时，复制到该分类目录。
- 文件名未命中任何分类时，复制到 `其他`。
- 文件名命中多个分类时，复制到 `待确认`。
- 非支持格式不参与关键词分类，统一复制到 `其他`。

示例：

| 文件名 | 命中结果 | 输出分类 |
| --- | --- | --- |
| `关于预算调整的请示.pdf` | 请示 | 请示 |
| `工作总结报告.docx` | 报告 | 报告 |
| `会议通知.png` | 通知 | 通知 |
| `技术标准.txt` | 非支持格式 | 其他 |
| `通知报告.pdf` | 通知、报告 | 待确认 |
| `未知材料.xlsx` | 非支持格式 | 其他 |

## 7. 分类配置

设置页新增“分类配置”区域。

### 7.1 可配置项

第一版允许用户维护分类列表。每个普通分类包含：

- 分类目录名
- 一个或多个关键词

用户可以：

- 新增普通分类。
- 编辑普通分类目录名。
- 编辑普通分类关键词。
- 删除普通分类。
- 删除 `其他` 和 `待确认`。

### 7.2 配置约束

- 新增普通分类必须至少包含一个关键词。
- 删除某个分类时，该分类的关键词一起删除。
- 删除后，文件名不会再命中该分类。
- `其他` 和 `待确认` 允许在设置中删除，但分类运行需要时会自动重建。
- 分类目录名和关键词保存前都执行清洗。
- 清洗后如果两个分类目录名重复，不允许保存。
- 清洗后如果两个关键词重复，不允许保存。
- 同一个关键词不能属于多个分类。
- 分类配置第一版只保存在本机设置中。
- 分类配置导入/导出作为后续扩展。

### 7.3 清洗规则

分类目录名和关键词保存前都自动清洗。

清洗目标：

- 替换 Windows 和 macOS 路径非法字符。
- 去除首尾空白。
- 压缩明显异常的连续空白。
- 避免空目录名或空关键词。

清洗后产生重复时，不自动追加序号、不自动合并，而是提示用户解决重复后再保存。

## 8. 后端设计

### 8.1 模块边界

新增 Rust 后端模块：

```text
src-tauri/src/classify.rs
```

该模块独立于现有标题识别链路，不调用：

- `extract`
- `scoring`
- 现有标题重命名逻辑
- 历史输出撤销逻辑

模块职责：

- 校验源文件夹。
- 递归扫描文件。
- 跳过隐藏文件和系统文件。
- 判断支持格式。
- 匹配分类规则。
- 创建带时间戳的输出目录。
- 创建分类子目录。
- 复制文件。
- 处理目标同名冲突。
- 汇总结果摘要。

### 8.2 数据结构建议

```rust
pub struct ClassificationSettings {
    pub categories: Vec<ClassificationCategory>,
}

pub struct ClassificationCategory {
    pub name: String,
    pub keywords: Vec<String>,
    pub system_kind: Option<SystemClassificationKind>,
}

pub enum SystemClassificationKind {
    Other,
    NeedsReview,
}
```

```rust
pub struct ClassifyFolderRequest {
    pub source_path: String,
    pub settings: ClassificationSettings,
}
```

```rust
pub struct ClassificationSummary {
    pub source_path: String,
    pub output_path: String,
    pub total_files: usize,
    pub copied_files: usize,
    pub failed_files: usize,
    pub category_counts: Vec<CategoryCount>,
    pub failures: Vec<ClassificationFailure>,
}

pub struct CategoryCount {
    pub category: String,
    pub count: usize,
}

pub struct ClassificationFailure {
    pub source_path: String,
    pub reason: String,
}
```

### 8.3 分类决策

```rust
pub enum ClassificationDecision {
    Category(String),
    Other,
    NeedsReview,
}
```

决策逻辑：

1. 如果文件扩展名不是当前支持格式，返回 `Other`。
2. 取文件名主体。
3. 对分类关键词做不区分大小写匹配。
4. 命中 0 个分类，返回 `Other`。
5. 命中 1 个分类，返回该分类。
6. 命中 2 个及以上分类，返回 `NeedsReview`。

## 9. IPC 设计

新增 Tauri 命令：

```text
classify_folder(source_path, classification_settings) -> ClassificationSummary
```

前端调用时传入：

- 用户选择的源文件夹路径。
- 当前分类配置快照。

返回时一次性得到分类摘要。第一版不提供分类进度事件。

前端执行期间显示处理中状态。命令完成后展示摘要。命令失败时展示结构化错误。

## 10. 前端设计

### 10.1 主界面入口

主界面新增按钮：

```text
分类文件夹
```

交互：

1. 用户点击按钮。
2. 打开文件夹选择对话框。
3. 用户选择源文件夹。
4. 前端读取当前分类配置。
5. 调用 `classify_folder`。
6. 执行期间按钮进入禁用或加载状态。
7. 完成后展示摘要。

第一版不做执行前预览。

### 10.2 摘要展示

摘要至少展示：

- 源文件夹路径。
- 输出文件夹路径。
- 参与分类的总文件数。
- 成功复制数。
- 各分类数量。
- 失败数量。
- 失败文件路径和原因。

隐藏文件和系统文件不进入摘要。

### 10.3 设置页

设置页新增“分类配置”区域。

功能：

- 展示分类列表。
- 新增分类。
- 编辑分类目录名。
- 编辑关键词。
- 删除分类。
- 保存时校验清洗后的目录名和关键词。

配置保存失败时，前端提示用户具体原因，例如：

- 分类名称不能为空。
- 关键词不能为空。
- 清洗后分类名称重复。
- 清洗后关键词重复。
- 新增分类至少需要一个关键词。

## 11. 错误处理

### 11.1 批次级失败

以下情况直接失败，不创建分类输出目录：

- 源路径不存在。
- 源路径不是文件夹。
- 源文件夹无读取权限。
- 输出目录创建失败。
- 分类配置无效且无法兜底。

### 11.2 文件级失败

以下情况记录为单文件失败，继续处理其他文件：

- 文件复制失败。
- 文件读取失败。
- 目标分类目录创建失败。
- 目标文件路径生成失败。

文件级失败进入 `ClassificationSummary.failures`。

### 11.3 同名冲突

如果目标分类目录中已经存在同名文件，系统自动追加序号：

```text
会议通知.pdf
会议通知 (2).pdf
会议通知 (3).pdf
```

不得覆盖已有文件。

移动或复制后的文件名保持原文件名不变，只在冲突时追加序号，并保持原扩展名大小写。

## 12. 本地与离线约束

分类功能必须完全离线运行。

第一版不得：

- 调用云端 API。
- 上传用户文件。
- 使用联网分类服务。
- 使用本地或云端 AI 模型进行分类。

分类判断只依赖本地配置和文件名规则。

## 13. 测试要求

### 13.1 后端测试

需要覆盖：

- 递归扫描源文件夹。
- 跳过隐藏文件和系统文件。
- 支持格式按关键词分类。
- 非支持格式进入 `其他`。
- 未命中关键词进入 `其他`。
- 多分类命中进入 `待确认`。
- 输出目录按时间戳创建。
- 不保留源目录层级。
- 同名冲突追加 `(2)`、`(3)`。
- 单文件复制失败后继续处理其他文件。
- 摘要统计正确。
- 分类配置清洗和重复校验。

### 13.2 前端测试

需要覆盖：

- 主界面展示“分类文件夹”入口。
- 点击入口后调用文件夹选择。
- 选择文件夹后调用 `classify_folder`。
- 执行期间显示处理中状态。
- 完成后展示摘要。
- 失败时展示错误信息。
- 设置页分类配置新增、编辑、删除和校验。

## 14. 验收样例

准备源目录：

```text
输入文件夹/
  关于预算调整的请示.pdf
  工作总结报告.docx
  会议通知.png
  技术标准.txt
  通知报告.pdf
  子目录/
    未知材料.xlsx
```

默认规则：

```text
请示 -> 请示
报告 -> 报告
通知 -> 通知
标准 -> 标准
```

执行分类后，在 `输入文件夹` 同级生成：

```text
Rustitler 分类输出 2026-07-03 1530/
  请示/
    关于预算调整的请示.pdf
  报告/
    工作总结报告.docx
  通知/
    会议通知.png
  待确认/
    通知报告.pdf
  其他/
    技术标准.txt
    未知材料.xlsx
```

说明：

- `技术标准.txt` 是非支持格式，因此进入 `其他`，即使文件名包含“标准”。
- `未知材料.xlsx` 是非支持格式，因此进入 `其他`。
- `通知报告.pdf` 同时命中“通知”和“报告”，因此进入 `待确认`。

摘要预期：

```text
总文件数：6
成功复制：6
失败数量：0

请示：1
报告：1
通知：1
标准：0
待确认：1
其他：2
```

## 15. 后续扩展

后续可考虑：

- 执行前预览分类计划。
- 分类批次进度事件。
- 取消正在执行的分类。
- 撤销本次分类复制结果。
- 将分类批次写入历史页面。
- 导入/导出分类配置。
- 手动确认 `待确认` 文件并由软件移动。
- 支持按正文内容或标题识别结果分类。
- 支持更复杂的规则，如正则、优先级、排除词。
- 支持用户选择输出目录。
- 支持保留源目录层级。
